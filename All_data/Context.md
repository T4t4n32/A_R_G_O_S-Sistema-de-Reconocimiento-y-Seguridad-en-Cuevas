# CALIBOTS KAIROS – Digital Platform

## FIRST LEGO League Challenge | UNEARTHED 2025–2026

---

## 1. Overview

This repository contains the **official digital platform** of **CALIBOTS KAIROS**, an educational robotics team from _Colegio Comfandi El Prado (Cali, Colombia)_, developed for the **FIRST LEGO League Challenge – UNEARTHED 2025–2026 season**.

The platform serves as:

- A **central presentation tool** for FLL judges during evaluations
    
- A **professional showcase** for external audiences (sponsors, media, community)
    
- A **digital legacy** documenting the team’s innovation, robot design, and core values
    

The website is designed to **optimize judging time**, clearly align with FLL rubrics, and function reliably in competition environments.

## 3. Strategic Objectives

The platform is built around four core objectives:

1. **Judging Efficiency**  
    Allow FLL judges to understand the team, project, and robot in under **2 minutes**, with optional deeper exploration.
    
2. **Rubric Alignment**  
    Explicitly demonstrate evidence for the three FLL pillars:
    
    - Innovation Project
        
    - Robot Design
        
    - Core Values
        
3. **Professional Image**  
    Present the team with a modern, credible, and competition-ready digital identity.
    
4. **Digital Legacy**  
    Preserve documentation, iterations, and learning for future seasons.
    

---

## 4. Platform Structure

The website follows a **Single Page Application (SPA)** architecture with smooth scrolling and a fixed navigation menu.

### Main Sections

1. **Home / About Us**
    
    - Hero video or animation
        
    - Team identity and season slogan
        
    - Interactive team cards
        
    - 4-year timeline
        
    - Coaches section
        
    - Core Values with real evidence
        
2. **Innovation Project**
    
    - Project overview and problem statement
        
    - Research and validation
        
    - Proposed solution
        
    - Embedded 3D model of the prototype
        
    - Iteration and development process
        
    - Technologies used
        
    - Expected impact
        
3. **Robot Design**
    
    - Design philosophy
        
    - Base robot (3D model)
        
    - Attachments gallery
        
    - Mission strategy table
        
4. **Multimedia Gallery**
    
    - Filterable photos and short videos
        
    - Categories by activity
        
5. **Kairos Play (Optional)**
    
    - Educational web-based mini-game
        
6. **Internal Area (Private)**
    
    - Changelog / Development log

## 5. Technology Stack

### Frontend

- **Next.js (React)**
    
- Responsive design (mobile-first)
    
- Smooth scroll SPA layout
    

### Deployment

- **Vercel** (recommended)
    

### Content Management

- **Markdown-based content**
    
- **Decap CMS** for non-technical updates
    

### 3D Visualization

- **Sketchfab embedded viewers**
    

### Media Optimization

- Compressed MP4 videos
    
- Optimized WebP images
    

### Offline Support

- Basic **Progressive Web App (PWA)** caching for competition environments
    

---

## 6. Performance Requirements

- Initial load time under **3 seconds** on mobile networks
    
- Optimized assets for low-bandwidth environments
    
- No heavy libraries without clear value
    

---

## 7. Content Update Workflow

1. Content is written or updated in Markdown
    
2. Media files are optimized before upload
    
3. Updates are logged in the internal changelog
    

### Changelog Format (Internal)

```
DD/MM – Description of change – Responsible person
```

Example:

```
15/10 – Added Mission 8 video to Robot section – Samuel
```

---

## 8. Printed Material Integration

The platform is designed to work in combination with a **Judges Dossier**:

- One or two-sided printed flyer
    
- Large QR codes linking to:
    
    - Main website
        
    - Kairos Play (if available)
        

The printed material is intentionally minimal and acts as a gateway to the digital platform.
